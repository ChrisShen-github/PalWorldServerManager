# Server Configuration Page Override

Use the existing teal Palworld control-center visual language from the dashboard and host setup pages instead of the master's purple palette.

- Treat this as a dense administration form: use category tabs, search, and progressive disclosure rather than rendering every field at once.
- Keep every field's Chinese label visible and show the raw INI key as secondary monospace metadata.
- Use a 3-column desktop grid, 2-column tablet grid, and 1-column mobile grid with no horizontal overflow.
- Mark destructive or high-load switches with a gold warning surface and explanatory helper text; never rely on color alone.
- Keep the single save action visible after long sections, but make it non-sticky on mobile.
- Preserve light/dark contrast parity, visible focus rings, 44px minimum controls, loading/disabled feedback, and reduced-motion support.
